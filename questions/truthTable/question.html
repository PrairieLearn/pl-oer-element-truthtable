Questions below were designed to showcase the different features of pl-truth-table The goal of pl-truth-table is simplify the rendering and grading of truth tables in engineering or computer science courses. Input columns are automatically generated and output columns are filled manually (for display or to grade student submissions).

Part 1

If the is-material attribute is set to true, tables are rendered statically and can be used as instructor-provided material for a question. In this example, 2 input-columns are auto-generated and 3 output columns are manually filled via the output-values attribute.
Part 2

For this table, is-material is set to false and the alphabet is set to TF. All output cells are prefilled with F so that students only have to change the relevant cells to T.
Part 3

This table provies no cell-level feedback and also allows the value X in outputs.

Part 4

The table uses different bit-widths for each input and output column. It also provides no feedback to students and is graded on an all-or-nothing basis.
