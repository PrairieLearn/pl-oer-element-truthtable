Questions below were designed to showcase the different features of pl-truth-table. The goal of pl-truth-table is simplify the rendering and grading of truth tables in math, engineering or computer science courses. Input columns are automatically generated and output columns are filled manually (for display or to grade student submissions).

Part 1

If the is-material attribute is set to true, tables are rendered statically and can be used as instructor-provided material for a question. In this example, 2 input-columns are auto-generated and 3 output columns are manually filled via the output-values attribute.
Part 2

For this table, is-material is set to false and the alphabet is set to TF. All output cells are prefilled with F so that students only have to change the relevant cells to T.
Part 3

This table has no prefill set, and show-partial-score is false, so it provides no cell-level feedback. It also has its alphabet set to TFX to allow the value X to be used for unknown outputs (that are based on unknown inputs, such as C).
Part 4

The table uses different bit-width values for each input column and also expects multiple bits for one of the output columns. Note that the size of the input box is determined automatically based on the expected output size. It also provides no cell-level feedback to students and is graded on an all-or-nothing basis because show-partial-score and show-percentage-score are both set to false.
Part 5

The table is identical to the previous one, but has constant-size set to a fixed value. Therefore, cells are not scaled to the size of the expected output and reveal less information to students. Note that placeholder and prefill should be empty in this case, so they don't give the expected size away for the students.
